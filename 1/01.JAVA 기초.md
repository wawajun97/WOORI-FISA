## 자바 실행 과정
  1. javac라는 컴파일러가 컴파일 -> .class 파일이 생성됨
  2. java라는 인터프리터로 실행
  
## 메모리 구조
  1. method 영역 : class 바이트 코드 적재, static 변수 적재
  2. heap 영역 : 동적으로 생성되는 객체 적재, GC 동작 위치
  3. stack 영역 : 로컬 변수 적재, 실제 연산과 로직 수행
  -> new 로 객체 생성 시 변수는 stack에 생성되고 실제 데이터는 heap에 저장
    ex) Car car = new Car("소나타");
        -> car는 stack에 저장됨(참조값이 저장 -> 주소는 아님)
        -> new Car("소나타")는 heap에 저장됨(참조값을 통하여 불러서 사용)
  
## 변수
  1. 선언 위치에 따른 구분
    1. 멤버 변수
      - class {} 내부에 선언되는 변수
      - class 기반으로 생성되는 객체의 구성 요소
      - heap 메모리에 생성
    2. 로컬 변수
      - 메소드, 생성자, (), {} 내부에 선언되는 변수
      - stack 메모리에 생성
      - 메소드/생성자 호출 시 생성 후 메소드/생성자 실행 종료 시 자동 소멸

2. 타입에 따른 구분
    1. 기본 타입(원시 타입)
        - char, byte, short, int, long, float, double, boolean
    2. 참조 타입
        - 기본 타입을 제외한 모든 타입
        - 객체 타입
